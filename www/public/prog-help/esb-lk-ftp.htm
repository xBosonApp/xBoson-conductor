LIST 模式说明

列表会返回到输出变量中, 变量的结构是一个数组, 每个数组元素是一个对象, 对象结构如下

{
  name : '文件名',
  type : '文件类型, d 是目录, - 是文件, l 是符号连接',
  date : '最后修改时间',
  size : '字节为单位的文件大小',
  rights : {'文件的操作权限'},
}

动态请求文件说明

如果希望 FTP 组件依据不同的情况请求不同的文件, 即前面的组件修改当前 FTP 请求的目标文件, 则在流中设置如下变量, 设置变量的方法可以使用 'JavaScript' 或 '变量' 来实现

  __ftp_change_file__
      -- 会替换掉配置时使用的文件名

  __ftp_change_mode__ 
      -- 'r' 配置中的文件名会被删除, 完全使用 __ftp_change_file__ 指定的文件名 (默认行为)
      -- 'a' __ftp_change_file__ 指定的文件名附加在配置中的目录路径后面 (配置中以 / 结尾的目录)
      -- 'f' 配置中是一个文件路径, 转换为一个目录路径, 并在后面附加 __ftp_change_file__
